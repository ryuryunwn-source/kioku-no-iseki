# 写し身（写真からカード生成）機能 — 実装メモ

新しいフォルダーの「写真→モンスターカード生成」機能を、ルールブック15章/19章の設計に沿って
『記憶の遺跡』に統合しました。**既存の固定48枚対戦は一切壊していません**（写し身を作らなければ従来通り）。

## やったこと（今夜・寝てる間）

### コア（プラグイン不要・エディタで即動く）
- `Scripts/GeneratedGuardianData.cs` … 19章の写し身データ＋`CardGenerator`（画像ハッシュ＆平均色から
  系統/コスト/攻防/技を**決定論的に**算出＝同じ写真は常に同じカード。15-2〜15-4）。オフライン命名候補も内蔵。
- `Scripts/GameTypes.cs` … `CardInstance` に `generated` を追加（19章）。写し身は合成`CardData`を
  `definition` に持つのでエンジンは固定カードと区別せず扱う（風化・転生・刻印・守護すべて共通）。
- `Scripts/GameEngine.cs` … `injectedWriteshi` を追加。対戦開始時、写し身の枚数だけ固定守護者を
  ランダムに置き換えて遺構デッキに合流させる（5章・13章末尾の方針）。
- `Scripts/GeneratedArt.cs` … 写真スプライトの実行時登録簿。
- `Scripts/PhotoWriteshi.cs` … 写真(Texture2D)→ハッシュ/HSV抽出→写し身生成→スプライト登録。`WriteshiCollection`（一時保管）。
- `GameUI.cs` / `OnlineController.cs` の `GetCardArt/GetArt` … `gen_` カードは登録写真を最優先で表示（オフ/オン両対応）。
- `GameUI.cs` … タイトルに「**写し身工房**」ボタン＋オーバーレイを追加。StartGameでコレクションを遺構デッキへ注入。

### 命名（AI・任意）
- `Scripts/WriteshiNamingService.cs` … Google Vision(ラベル)→Groq(LLM)で日本語名を生成。
  失敗/キー無し/オフラインなら自動で候補名にフォールバック（対戦は必ず成立）。
- APIキーは **`Assets/Resources/writeshi_api.txt`（.gitignore済み＝GitHubに公開しない）** から読み込み。
  1行目=Vision、2行目=Groq。テンプレは `writeshi_api.txt.template`。
  ※新フォルダーに直書きされていたキーはこのローカルファイルに移し、リポジトリには載せていません。

## 今すぐ試せること（エディタ・実機不要）
1. Play → タイトルの「写し身工房」→「**テスト写し身を追加（画像なし）**」を数回押す。
2. 「閉じる」→「AIと対戦」。ランダム画像から作った写し身が遺構デッキに混ざって対戦に登場します
   （カード名・系統・攻防・技はすべて生成値。同じ画像なら毎回同じカード）。

## 残タスク（あなたの確認・操作が要る部分）
1. **実機のカメラ/アルバム**：導入**済み**。
   - `Assets/Plugins/NativeCamera` `Assets/Plugins/NativeGallery` をコピー済み。
   - `KiokuNoIseki.asmdef` に `NativeCamera.Runtime` / `NativeGallery.Runtime` を参照追加済み。
   - `ProjectSettings.asset` の Scripting Define Symbols に **`KIOKU_NATIVE_MEDIA`**（Android/Standalone）を追加済み。
   - → Unity起動でインポート＆再コンパイルされ、工房の「アルバムから選ぶ」「カメラで撮る」が有効化されます。
   - Editorでは「アルバムから選ぶ」がファイル選択ダイアログにフォールバック（NativeGalleryのEditor動作）。
     「カメラで撮る」は実機のみ。実機は初回にカメラ/写真アクセス許可を求めます。
2. **AI命名を使うなら**：`writeshi_api.txt` のキーが有効か確認（工房画面に「AI命名: 有効/無効」を表示）。
   配布時はキー直埋め非推奨（15-7）。個人利用なら現状でOK。
3. **オンライン対戦**：写し身の**描画**は対応済みだが、ネット同期（写真は送らず結果だけ・18-4）は未実装。
   まずオフライン/AI戦で確認するのがおすすめ。
4. **新規スクリプトの .meta** はUnityが初回起動時に自動生成します。
5. **置き換え枚数**：現状は「作った写し身の数だけ」固定守護者を置換（可変）。上限は守護者30体。

## 設計判断メモ
- 「簡易版(2)で動かす／データ構造は19章通り」という指示に沿った実装。ステータス生成は15章の
  決定論アルゴリズム簡易版まで入れてある（ダミー固定値ではない）。
- 既存ビルドを壊さない事を最優先し、プラグイン依存コードは `#if KIOKU_NATIVE_MEDIA` で隔離。
